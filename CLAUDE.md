# Claude Code — exobrain-specific guidance

@AGENTS.md

This file holds Claude Code-specific guidance. The shared conventions in `AGENTS.md` apply to every agent; the notes below apply only when running under Claude Code.

## Tooling primitive names

When `AGENTS.md` or a skill refers to "the agent's primitive for X", the Claude Code names are:

- **Skill discovery / invocation** — the `Skill` tool. `tier: always` skills auto-load; `tier: optional` skills are listed in `optional-skills.md` and read on demand.
- **File ops** — `Read`, `Edit`, `Write`. Prefer `Edit` over `Bash sed`.
- **Search** — `Grep`, `Glob`. Prefer these over `Bash find`.
- **Task tracking** — `TaskCreate` / `TaskUpdate` for multi-step work.

## Git history hygiene

Keep this repo's history agent-neutral — omit Claude Code's default attribution. No `🤖 Generated with Claude Code` footer in PR bodies, and no `Co-Authored-By: Claude …` trailer in commit messages.

## Auto-loading

Claude Code auto-loads two CLAUDE.md files:

1. **This file** at the repo root — handcrafted, Claude-specific guidance. Its `@AGENTS.md` import pulls in the global scope.
2. **`.claude/CLAUDE.md`** — generated by `connect-agent.sh claude`; `@-import`s two generated files: `.claude/connected-scopes.md` (a manifest of `@-import`s to each connected deeper scope's live `AGENTS.md`/`CLAUDE.md`, shallow→deep — referenced, not copied, so scope edits show up without a relink) and `.claude/optional-skills.md` (the agent-filtered optional-skills index).

Each agent gets its **own** generated surface; no two agents write the same file. The manifest references source specs by relative path so a copied checkout (test sandbox, worktree) still resolves them. The `@<file>.md` directive is Claude-specific: Codex instead reads a generated in-repo `AGENTS.override.md` (which natively outranks `AGENTS.md`), and OpenClaw gets the same context inlined into its `USER.md` via marker-block injection.
