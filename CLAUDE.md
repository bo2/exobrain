## Tooling primitive names

When `AGENTS.md` or a skill refers to "the agent's primitive for X", the Claude Code names are:

- **Skill discovery / invocation** ‚Äî the `Skill` tool. `tier: always` skills auto-load; `tier: optional` skills are listed in `optional-skills.md` and read on demand.
- **File ops** ‚Äî `Read`, `Edit`, `Write`. Prefer `Edit` over `Bash sed`.
- **Search** ‚Äî `Grep`, `Glob`. Prefer these over `Bash find`.
- **Task tracking** ‚Äî `TaskCreate` / `TaskUpdate` for multi-step work.

## Auto-loading

Claude Code auto-loads two CLAUDE.md files:

1. **This file** at the repo root ‚Äî handcrafted, Claude-specific guidance.
2. **`.claude/CLAUDE.md`** ‚Äî generated by `connect-agent.sh claude`; a single `@-import` of `.claude/AGENTS.override.md`, the composed file holding every connected scope's spec (shallow‚Üídeep) plus the optional-skills index.

The `@<file>.md` directive is Claude-specific; Codex and OpenClaw use their own injection mechanisms.
